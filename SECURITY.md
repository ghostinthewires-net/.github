# Security Policy

## Reporting a Vulnerability

To report a security vulnerability in any ghostinthewires project, please
email gitw@ghostinthewires.net using the encryption key below.

Do not open a public issue for security vulnerabilities.

**Encryption key:** https://ghostinthewires.net/.well-known/pgp-key.asc
**Fingerprint:** F90D E5A4 2657 59CA 4CB3 3F2D 8CF1 8303 85E9 3B4F

Encrypting your report ensures only the maintainer can read it. See
[KEYS.md](KEYS.md) for full key documentation.

## Response Time

This is a volunteer-maintained open source project. We will make a
reasonable effort to respond within 14 days. Critical vulnerabilities
affecting user security will be prioritized.

## Scope

Security reports are accepted for:
- ghostinthewires/netrunner — hardened workstation installer
- ghostinthewires/hardline — Raspberry Pi network infrastructure
- ghostinthewires/handbook — documentation errors that could lead users
  to insecure configurations

## Disclosure Policy

We follow coordinated disclosure. Please allow reasonable time to
investigate and patch before public disclosure. We will credit reporters
in the relevant release notes unless anonymity is requested.
